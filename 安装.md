# 安装与部署指南

## 前置条件

- Node.js 18+
- MySQL 8.0+
- AWS 账号（用于 S3 文件存储）

---

## 第一步：安装依赖

```bash
npm install
```

---

## 第二步：配置环境变量

复制示例文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入以下内容：

```env
# 数据库（替换为你的 MySQL 连接信息）
DATABASE_URL="mysql://root:你的密码@localhost:3306/inspection_db"

# JWT 密钥（随意填一个长字符串）
JWT_SECRET="your-super-secret-jwt-key-change-this-in-production"
JWT_EXPIRES_IN="7d"

# AWS S3（第三步完成后填入）
AWS_REGION="ap-southeast-1"
AWS_ACCESS_KEY_ID="填入你的 Access Key ID"
AWS_SECRET_ACCESS_KEY="填入你的 Secret Access Key"
AWS_S3_BUCKET="inspection-platform-attachments"

# 环境
NODE_ENV="development"
```

---

## 第三步：配置 AWS S3

### 3.1 创建 S3 Bucket

1. 登录 [AWS 控制台](https://console.aws.amazon.com)
2. 搜索并进入 **S3**
3. 点击 **"创建存储桶"**
4. 填写：
   - 存储桶名称：`inspection-platform-attachments`（或自定义，需与 `.env` 中 `AWS_S3_BUCKET` 一致）
   - 区域：选 **亚太区域（新加坡）ap-southeast-1**（或其他，需与 `.env` 中 `AWS_REGION` 一致）
   - **取消勾选** "阻止所有公共访问"（允许预签名 URL 访问）
5. 点击 **"创建存储桶"**

### 3.2 配置 CORS

进入刚创建的 Bucket → **权限** → **跨源资源共享 (CORS)** → 编辑，粘贴：

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["http://localhost:3000"],
    "ExposeHeaders": ["ETag"]
  }
]
```

### 3.3 创建 IAM 用户

1. 进入 **IAM** → **用户** → **创建用户**
2. 用户名填：`inspection-app`
3. **不勾选** "提供用户对 AWS 管理控制台的访问权限"（程序访问不需要）
4. 点击 **"下一步"**

### 3.4 附加权限策略

在权限页面：

1. 选择 **"直接附加策略"**
2. 点击 **"创建策略"** → 选择 **JSON** 标签页
3. 粘贴以下最小权限策略（替换 `YOUR_BUCKET_NAME` 为你的 Bucket 名称）：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    },
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME"
    }
  ]
}
```

4. 策略名称填：`inspection-s3-policy`，点击 **"创建策略"**
5. 回到用户创建页面，搜索并勾选 `inspection-s3-policy`
6. 点击 **"下一步"** → **"创建用户"**

### 3.5 获取 Access Key

1. 进入刚创建的用户 → **安全凭证** 标签页
2. 点击 **"创建访问密钥"**
3. 用例选 **"在 AWS 外部运行的应用程序"**
4. 点击 **"下一步"** → **"创建访问密钥"**
5. **立即复制** Access Key ID 和 Secret Access Key（页面关闭后无法再查看 Secret）
6. 填入 `.env` 文件对应字段

## 第四步：初始化数据库

确保 MySQL 已启动，并创建数据库：

```sql
CREATE DATABASE inspection_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

推送 Prisma Schema 到数据库：

```bash
npx prisma db push
```

填充测试数据：

```bash
npm run db:seed
```

## API 接口一览

### 认证
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 注册 |
| POST | `/api/auth/login` | 登录，返回 JWT Token |

### 任务管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tasks` | 任务列表（支持过滤） |
| POST | `/api/tasks` | 创建任务 |
| GET | `/api/tasks/:id` | 任务详情 |
| DELETE | `/api/tasks/:id` | 删除任务（仅 hq_admin） |
| PATCH | `/api/tasks/:id/status` | 状态流转 |

### 整改
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tasks/:id/rectifications` | 提交整改（仅 store_manager） |
| GET | `/api/tasks/:id/rectifications` | 查看整改记录 |

### 文件
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/files/presign-upload` | 获取 S3 上传预签名 URL |
| GET | `/api/files/:id/presign-download` | 获取 S3 下载预签名 URL |

## 常见问题

**Q: `prisma db push` 报错连接失败**
检查 `.env` 中 `DATABASE_URL` 的用户名、密码、端口是否正确，以及 MySQL 服务是否已启动。

**Q: S3 上传报错 `InvalidAccessKeyId`**
检查 `.env` 中 `AWS_ACCESS_KEY_ID` 和 `AWS_SECRET_ACCESS_KEY` 是否正确填写，注意不要有多余空格。

**Q: S3 上传报错 `NoSuchBucket`**
检查 `.env` 中 `AWS_S3_BUCKET` 与实际创建的 Bucket 名称是否完全一致，区域是否匹配。

**Q: 登录后请求报 401**
请求头需要带 `Authorization: Bearer <token>`，token 从登录接口返回。
