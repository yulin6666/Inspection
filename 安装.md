# Installation & Deployment Guide

## Prerequisites

- Node.js 18+
- MySQL 8.0+
- AWS account (for S3 file storage)

---

## Step 1: Install Dependencies

```bash
npm install
```

---

## Step 2: Configure Environment Variables

Copy the example file:

```bash
cp .env.example .env
```

Edit `.env` and fill in the following:

```env
# Database (replace with your MySQL connection info)
DATABASE_URL="mysql://root:your_password@localhost:3306/inspection_db"

# JWT secret (use any long random string)
JWT_SECRET="your-super-secret-jwt-key-change-this-in-production"
JWT_EXPIRES_IN="7d"

# AWS S3 (fill in after completing Step 3)
AWS_REGION="ap-southeast-1"
AWS_ACCESS_KEY_ID="your-access-key-id"
AWS_SECRET_ACCESS_KEY="your-secret-access-key"
AWS_S3_BUCKET="inspection-platform-attachments"

# Environment
NODE_ENV="development"
```

## Step 3: Configure AWS S3

### 3.1 Create an S3 Bucket

1. Log in to the [AWS Console](https://console.aws.amazon.com)
2. Search for and open **S3**
3. Click **"Create bucket"**
4. Fill in:
   - Bucket name: `inspection-platform-attachments` (or custom — must match `AWS_S3_BUCKET` in `.env`)
   - Region: **Asia Pacific (Singapore) ap-southeast-1** (or other — must match `AWS_REGION` in `.env`)
   - **Uncheck** "Block all public access" (required for presigned URL access)
5. Click **"Create bucket"**

### 3.2 Configure CORS

Open the bucket → **Permissions** → **Cross-origin resource sharing (CORS)** → Edit, paste:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["http://localhost:3000"],
    "ExposeHeaders": ["ETag"]
  }
]
```

### 3.3 Create an IAM User

1. Go to **IAM** → **Users** → **Create user**
2. Username: `inspection-app`
3. **Do not check** "Provide user access to the AWS Management Console" (not needed for programmatic access)
4. Click **"Next"**

### 3.4 Attach a Permission Policy

On the permissions page:

1. Select **"Attach policies directly"**
2. Click **"Create policy"** → select the **JSON** tab
3. Paste the following least-privilege policy (replace `YOUR_BUCKET_NAME` with your bucket name):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    },
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME"
    }
  ]
}
```

4. Policy name: `inspection-s3-policy`, click **"Create policy"**
5. Return to the user creation page, search for and select `inspection-s3-policy`
6. Click **"Next"** → **"Create user"**

### 3.5 Get Access Keys

1. Open the newly created user → **Security credentials** tab
2. Click **"Create access key"**
3. Use case: **"Application running outside AWS"**
4. Click **"Next"** → **"Create access key"**
5. **Copy immediately** — the Secret Access Key cannot be viewed again after closing the page
6. Paste both values into the corresponding fields in `.env`

## Step 4: Initialize the Database

Make sure MySQL is running, then create the database:

```sql
CREATE DATABASE inspection_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Push the Prisma schema to the database:

```bash
npx prisma db push
```

Seed test data:

```bash
npm run db:seed
```

On success you should see:


## Quick Start (Full Flow)

Run these commands in order from a fresh clone:

```bash
# 1. Install dependencies
npm install

# 2. Configure environment variables (edit .env with your DB and S3 info)
cp .env.example .env

# 3. Push database schema
npx prisma db push

# 4. Seed test data
npm run db:seed

# 5. Start development server
npm run dev
```

After the server starts, visit [http://localhost:3000](http://localhost:3000) and log in with:

```
HQ Admin:      admin@test.com      password: password123
Inspector:     inspector@test.com  password: password123
Store Manager: manager@test.com    password: password123
```

## API Reference

### Auth
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/register` | Register |
| POST | `/api/auth/login` | Login, returns JWT token |

### Task Management
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/tasks` | Task list (supports filtering) |
| POST | `/api/tasks` | Create task |
| GET | `/api/tasks/:id` | Task detail |
| DELETE | `/api/tasks/:id` | Delete task (hq_admin only) |
| PATCH | `/api/tasks/:id/status` | Status transition |

### Rectification
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/tasks/:id/rectifications` | Submit rectification (store_manager only) |
| GET | `/api/tasks/:id/rectifications` | View rectification records |

### Files
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/files/presign-upload` | Get S3 presigned upload URL |
| GET | `/api/files/:id/presign-download` | Get S3 presigned download URL |

### Other
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/audit-logs` | Audit logs (hq_admin only) |
| GET | `/api/dashboard/metrics` | Operations dashboard metrics |

## Troubleshooting

**Q: `prisma db push` fails with a connection error**
Check that the username, password, and port in `DATABASE_URL` are correct and that MySQL is running.

**Q: S3 upload fails with `InvalidAccessKeyId`**
Verify that `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` in `.env` are correct with no extra whitespace.

**Q: S3 upload fails with `NoSuchBucket`**
Check that `AWS_S3_BUCKET` in `.env` exactly matches the bucket name you created, including the region.

**Q: Requests return 401 after login**
Requests must include the header `Authorization: Bearer <token>` where the token is returned from the login endpoint.
